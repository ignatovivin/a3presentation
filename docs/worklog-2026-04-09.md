## Что изменилось в backend

- Добавлен общий модуль [chart_render_contract.py](../src/a3presentation/services/chart_render_contract.py).
  Он централизует visible-series semantics, combo fallback, primary/secondary axis, axis number formats и безопасность transpose.
- [table_chart_analyzer.py](../src/a3presentation/services/table_chart_analyzer.py) теперь строит варианты графиков через `variant_label`:
  `Сравнение`, `Единичный`, `Комбинированный`.
- Safe mixed-unit сценарии вроде `RUB + %` больше не сводятся к одному unsafe single-axis графику.
  Для combo-кандидатов процентная серия может уходить на secondary value axis.
- Analyzer теперь умеет выводить unit из semantic header вроде `Доля`, `Маржа`, `Процент`, `share`, `rate`, даже если сами ячейки не содержат `%`.
- [pptx_generator.py](../src/a3presentation/services/pptx_generator.py) использует общий chart render contract и умеет рендерить secondary value axis для mixed-unit combo.
- [deck_audit.py](../src/a3presentation/services/deck_audit.py) использует тот же chart render contract, проверяет secondary axis и сравнивает chart/footer geometry с template slot там, где это нужно для prototype templates.
- [layout_capacity.py](../src/a3presentation/services/layout_capacity.py) расширен в сторону placeholder-aware capacity derivation.
- [planner.py](../src/a3presentation/services/planner.py) продвинут по dense continuation:
  `dense_text_full_width` используется не только для чистого narrative, но и для paragraph-dominant mixed narrative + bullets.

Главные зафиксированные правила:

- chart formatting должен опираться на `python-pptx` и Open XML documentation, а не на локальную эвристику
- axis labels и data labels это разные контракты
- `%` допустим только на оси, где все series процентные
- mixed `number/RUB + %` должен идти через combo + secondary axis
- generator и deck audit должны использовать один общий chart render contract
- frontend preview и backend generation должны проверяться как единый pipeline
- каждая новая задача должна начинаться с анализа правил в `docs`

## Текущее состояние после 9 апреля

- Проект сильнее сдвинулся от набора отдельных фиксов к контрактной системе:
  analyzer, UI, preview, planner, generator и deck-audit теперь связаны общими chart semantics.
- Template-aware audit продвинут за пределы обычных text placeholders:
  footer, chart slot и auxiliary placeholders теперь участвуют в runtime quality checks.
- Dense continuation стал не только текстовым оптимизатором, но и частью quality-backed layout contract.
- Основной следующий шаг остается parity hardening:
  один и тот же `ChartSpec` должен прослеживаться через preview controls, `chart_overrides` request, plan response, generated deck и `deck_audit`.
